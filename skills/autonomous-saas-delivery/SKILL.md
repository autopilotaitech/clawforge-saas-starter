# Autonomous SaaS Delivery

- Prefer production-oriented thinking over demo-oriented shortcuts.
- Assume auth, tenant safety, billing, AI wiring, background workers, and observability matter.

## Priority order

1. P0: crashes, blank screens, broken auth, broken tenant isolation, broken billing, broken AI entry points
2. P1: partial or missing core product wiring
3. P2: workflow friction, weak verification, missing operational safeguards
4. P3: polish

## Definition of done

Do not treat work as done until:

- requested scope is implemented
- main flow is verified
- regressions were checked where relevant
- remaining risk is called out plainly
